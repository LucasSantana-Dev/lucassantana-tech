# Issue Tracker

**Tracker:** GitHub Issues (`LucasSantana-Dev/lucassantana-tech`)  
**CLI:** `gh` (GitHub CLI)  
**External PRs as triage surface:** No

## How skills use this

- `/to-issues` — creates issues via `gh issue create`
- `/triage` — reads and labels issues via `gh issue list` / `gh issue edit`
- `/to-prd` — publishes PRDs as GitHub issues (pinned or labeled)
- `/qa` — links test findings to issues

## Label conventions

See `docs/agents/triage-labels.md` for the triage label vocabulary.

## Notes

This is a personal portfolio repo. Issues track planned features, design iteration tasks, and performance work. External contributors are not expected; PRs are not used as a request surface.
